# Руководство пользователя — Поиск по базе знаний университета

Система позволяет загрузить документы (PDF/DOCX) и выполнять полнотекстовый
поиск по их содержимому с подсветкой найденных фрагментов.

> **Статус документации.** Гайд описывает пользовательские сценарии согласно
> контракту интерфейса (`CONTRACTS.md`). Скриншоты будут добавлены после
> подключения интерактивного фронтенда (стадии FE-02/03/04/07) — до этого UI
> отрисовывает статичный скелет, и снимки экрана были бы неактуальны.

## 1. Начало работы

1. Откройте приложение в браузере: `http://localhost:8080`.
2. Вверху страницы — заголовок «Поиск по базе знаний» и две вкладки:
   **Загрузка** и **Поиск**. По умолчанию открыта вкладка «Загрузка».

## 2. Загрузка документов

Вкладка **Загрузка**.

1. Перетащите файл в область «Перетащите PDF/DOCX сюда» или нажмите на неё,
   чтобы выбрать файл на диске.
2. Загруженный документ появится в списке ниже с индикатором статуса.

**Поддерживаемые форматы и ограничения:**

| Условие | Результат |
|---|---|
| PDF (`application/pdf`) | принимается |
| DOCX (`application/vnd.openxmlformats-officedocument.wordprocessingml.document`) | принимается |
| Другой тип файла (txt, png, zip…) | ошибка: «Unsupported file type» |
| Пустой файл (0 байт) | ошибка: «Empty file» |
| Размер больше 20 МБ | ошибка: «File exceeds 20 MB limit» |

**Статусы документа** (подпись в списке):

| Статус | Подпись | Значение |
|---|---|---|
| `uploaded` | Загрузка… | файл принят, ожидает обработки |
| `indexing` | Индексация… | идёт извлечение текста и индексация |
| `indexed` | Готово | документ доступен для поиска |
| `error` | Ошибка | обработка не удалась |

Дождитесь статуса **«Готово»** — только после этого документ участвует в поиске.

## 3. Поиск

Вкладка **Поиск**.

1. Введите запрос в поле «Введите запрос».
2. Нажмите кнопку **«Искать»**.
3. Результаты отображаются карточками. Каждая карточка содержит:
   - **имя файла** — из какого документа найден фрагмент;
   - **страницу** — номер страницы (например, «стр. 1»);
   - **релевантность** — числовая оценка соответствия запросу;
   - **фрагмент текста** с **подсветкой** совпадений (выделены в тексте).

Если ничего не найдено, показывается сообщение **«Ничего не найдено»**.

### Постраничная навигация

Под результатами — панель навигации по страницам. По умолчанию выдаётся до
10 результатов на страницу; общее число совпадений используется для расчёта
количества страниц.

## 4. Типовые проблемы

| Симптом | Причина / решение |
|---|---|
| Документ не находится в поиске | Проверьте, что его статус — «Готово», а не «Индексация…». |
| Файл не загружается | Проверьте формат (только PDF/DOCX) и размер (до 20 МБ). |
| Пустой результат по очевидному запросу | Документ ещё индексируется либо текст в нём не распознан (например, скан без текстового слоя). |

## 5. Для QA / проверки на чистом стенде

Воспроизвести сценарий целиком:

```bash
docker compose up -d --wait          # поднять стек (app, postgres, ES, redis, front)
open http://localhost:8080           # открыть UI
# Загрузить tests/fixtures/ok.pdf → дождаться «Готово» → искать «knowledge base»
```

Автотест этого пути — `e2e/tests/critical-path.spec.ts` (QA-02).
