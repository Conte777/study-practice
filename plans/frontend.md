# План: Frontend (FE)

**Роль:** пользовательский интерфейс загрузки и поиска, клиент-серверное взаимодействие.
**Стек:** TypeScript, React 18 + Vite, fetch/axios, отдача статики через Nginx (сборка для DevOps).
**Директория:** `frontend/` (components/, pages/, services/, App.tsx).

Стадии по фичам. Гейт = ① runnable-проверка (dev-сервер/тест) ② чеклист Done (ID ТЗ) ③ ревью sub-agent'ом.

Ревью гейта:
> Task(subagent_type="review-code", prompt="Проверь стадию N плана frontend: соответствие <ID>, состояния загрузки/ошибок, доступность, отсутствие лишних ре-рендеров. Верни confirmed-findings.")

---

## Стадия 1 — Каркас + сервисный слой

**Задачи**
- `npm create vite@latest -- --template react-ts`.
- `src/services/api.ts` — обёртки над `/api/v1/documents/upload` и `/api/v1/search` (baseURL из env).
- Роутинг/лейаут страниц (загрузка + поиск), общий стиль.

**Гейт 1**
- Команда: `npm run build && npm run dev` — собирается, страница открывается без ошибок в консоли.
- Команда: `npm run lint` (eslint) — чисто.
- Чеклист: [ ] TS strict [ ] api.ts типизирован [ ] baseURL из env [ ] сборка проходит.
- Ревью: sub-agent review-code — типы ответов API совпадают с контрактом BE; обработка сетевых ошибок в services.

## Стадия 2 — Загрузка документов (FE-01, FE-02, FE-03)

**Задачи**
- Drag-and-Drop зона с множественной загрузкой (FE-01).
- Прогресс-бар на каждый файл, состояния: `Загрузка… → Индексация… → Готово / Ошибка` (FE-02).
- Список загруженных: название, дата, статус (FE-03).

**Гейт 2**
- Команда: `npm run dev`, перетащить 2 файла → у каждого прогресс проходит все состояния, оба появляются в списке.
- Кейс ошибки: загрузить .txt → карточка в состоянии `Ошибка` с текстом от BE (400).
- Чеклист: [ ] FE-01 DnD + мультизагрузка [ ] FE-02 все 4 состояния [ ] FE-03 список с датой/статусом [ ] ошибка сервера отображается.
- Ревью: sub-agent review-code — гонки при параллельной загрузке; статус «Индексация» синхронизирован с BE; утечки при отмене.

## Стадия 3 — Поиск: ввод + карточки (FE-04, FE-05)

**Задачи**
- Поле ввода + кнопка «Найти»; запуск по клику и по Enter (FE-04).
- Результаты — карточки: имя файла, номер страницы, фрагмент, оценка релевантности (FE-05).

**Гейт 3**
- Команда: `npm run dev`, ввести запрос → Enter и кнопка дают одинаковый результат; карточки со всеми 4 полями.
- Чеклист: [ ] FE-04 Enter + кнопка [ ] FE-05 все поля карточки [ ] loading-состояние во время запроса.
- Ревью: sub-agent review-code — debounce/дабл-сабмит; экранирование текста фрагмента (XSS до подсветки).

## Стадия 4 — Подсветка совпадений (FE-06)

**Задачи**
- Слова из запроса в фрагменте — жёлтый фон (`<mark>`), без ломки вёрстки.
- Источник подсветки: highlight от ES (согласовать с BE) либо клиентская подсветка по токенам запроса.

**Гейт 4**
- Команда: поиск по слову → в карточках именно совпавшие слова выделены жёлтым, регистронезависимо.
- Чеклист: [ ] FE-06 подсветка есть [ ] безопасно рендерится (без innerHTML-инъекций) [ ] работает для кириллицы/морфологии в разумных пределах.
- Ревью: sub-agent review-security — рендер подсветки не открывает XSS; санитайзинг фрагмента.

## Стадия 5 — Пагинация / бесконечный скролл (FE-07)

**Задачи**
- По 10 результатов на страницу: пагинация **или** бесконечный скролл (загрузка при скролле вниз).
- Синхронизация с BE (from/size).

**Гейт 5**
- Команда: запрос с >10 результатами → видно 10, подгрузка следующих работает, дублей нет.
- Чеклист: [ ] FE-07 порционная выдача по 10 [ ] нет повторной загрузки одних и тех же [ ] индикатор дозагрузки.
- Ревью: sub-agent review-code — корректность offset при быстром скролле; отсутствие бесконечного цикла запросов.

## Стадия 6 — Пустой результат + адаптив (FE-08, FE-09)

**Задачи**
- Нет результатов → «По вашему запросу ничего не найдено. Попробуйте изменить формулировку» (FE-08).

**Гейт 6 (финальный)**

- Команда: DevTools responsive — проверить 320 / 768 / 1920px, горизонтального скролла нет.
- Чеклист: [ ] FE-08 текст точный [ ] FE-09 три брейкпоинта ок [ ] lint/build чисто.
- Ревью: sub-agent review-architecture — переиспользование компонентов, отсутствие дублирования стилей, разумная структура pages/components.
