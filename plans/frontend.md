**Директория:** `frontend/`

## Стадия 1 — Прогресс-бар (FE-02)

**Контекст:** статус индексации на бэке дискретный (`uploaded → indexing → indexed → error`),
процента нет → бар индетерминированный (анимированная полоска на время pending).

**Задачи**
- В `src/pages/UploadPage.tsx` в каждом `<li>` добавить `<progress>` или `div.progress-bar`:
  - `uploaded`/`indexing` → индетерминированный бар (`<progress>` без `value`, или CSS-анимация ширины);
  - `indexed` → бар заполнен 100%, зелёный;
  - `error` → бар красный/скрыт, показать текст ошибки.
- Стили в `src/App.css`: `.progress`, `.progress--done`, `.progress--error`.
- Сохранить существующие `data-testid` (`upload-status`), добавить `data-testid="upload-progress"` для E2E.

**Гейт 1**
- Загрузить файл → видно движущийся бар на время `Загрузка…/Индексация…`, затем 100% «Готово».
- Ошибочный файл → бар в состоянии ошибки + текст.
- `npm run build` — ок.
- Чеклист: [ ] бар для всех 4 состояний [ ] тексты статусов остались [ ] testid не сломаны.

## Стадия 3 — Авторизация (логин/пароль, без ролей)

**Зависимость:** backend-стадия auth (`POST /auth/login`, `/auth/register`, `Bearer`-токен).

**Задачи**
- `src/pages/LoginPage.tsx` — форма username/password, кнопки «Войти» / «Регистрация».
- `src/services/auth.ts` — `login`/`register`/`logout`; токен хранить в `localStorage`.
- В `src/services/api.ts` — добавлять `Authorization: Bearer <token>` ко всем защищённым запросам;
  на `401` — сбрасывать токен и редиректить на логин.
- В `App.tsx` — гейт: нет токена → показывать `LoginPage`, иначе вкладки Загрузка/Поиск + кнопка «Выйти».

**Гейт 3**
- Без токена приложение показывает экран входа.
- Логин demo-пользователем → доступ к загрузке/поиску; запросы уходят с `Bearer`.
- `401` от API → возврат на логин.
- Чеклист: [ ] логин/регистрация работают [ ] токен в localStorage [ ] header проставляется [ ] logout чистит токен [ ] 401 обрабатывается.

## Стадия 4 — История поиска (выпадашка)

**Зависимость:** backend `GET /api/v1/search/history?limit=N`.

**Задачи**
- В `src/services/api.ts` — метод `getSearchHistory(limit)`.
- В `SearchPage.tsx` — выпадающий список под строкой поиска:
  - грузить историю при фокусе на инпуте (и обновлять после успешного поиска);
  - показывать последние N запросов (текст + число результатов, опц. дата);
  - клик по элементу → подставляет запрос в инпут и сразу запускает `runSearch`;
  - закрытие по blur/Escape/клику вне (учесть, чтобы клик по элементу успел отработать до blur — `onMouseDown`).
- Стили `.search-history` в `App.css`; учесть адаптив (Стадия 2) — на 320px не вылезать за viewport.
- `data-testid="search-history"` / `search-history-item` для E2E.

**Гейт 4**
- Сделать поиск → повторный фокус на инпуте показывает его в выпадашке.
- Клик по записи → запрос выполняется повторно.
- Пустая история → выпадашка не показывается (или показывает «пусто»).
- Чеклист: [ ] грузится из API [ ] клик повторяет запрос [ ] закрытие по blur/Esc/вне [ ] адаптив ок [ ] testid есть.
